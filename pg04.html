  • Relacionamenttos entre Modelos

  • Uma relação 1:N significa que um registro de uma tabela pode estar associado a vários registros de outra tabela
  • Relações 1:n permitem conectar modelos de forma lógica
  • HasMany e Belongsto são essencias para configurar associações

Descrição - Vídeo 02

  • BACK-END

  • O back-end é a parte de aplicações que gerencia a lógica de negócios, processa dados e integra com o banco de dados.
  • Ele é o responsavél por manter a funcionalidade por trás da interface que o usuário vê no front-end.
